# 策略配置系统使用指南

## 概述

AI-OKX-Trader 支持通过 YAML 配置文件自定义交易策略，无需修改代码即可切换不同的交易策略。策略配置文件位于 `strategies/` 目录下，通过 `.env` 文件中的 `STRATEGY_NAME` 参数进行切换。

## 快速开始

### 1. 选择策略

在 `.env` 文件中设置 `STRATEGY_NAME` 参数：

```bash
# 15分钟短线趋势跟随策略（默认）
STRATEGY_NAME=15m_trend_following

# 或者选择其他策略
# STRATEGY_NAME=5m_scalping        # 5分钟超短线剥头皮策略
# STRATEGY_NAME=1h_swing           # 1小时波段交易策略
```

### 2. 重启程序

修改策略后，重启交易程序即可生效：

```bash
# Windows
start.bat

# Linux/Mac
./start.sh
```

## 内置策略说明

### 1. 15m_trend_following（15分钟短线趋势跟随）

**文件**: `strategies/15m_trend_following.yaml`

**特点**:
- 主周期: 15分钟
- 趋势判断: 1小时定方向
- 入场精度: 5分钟
- 仓位范围: 50%-100%
- 止损范围: 0.5%-1.0% (基于1H ATR)
- 止盈目标: 1.0%-2.0%
- 盈亏比: 最低 1:1.5
- 持仓时间: 1-2小时

**适用场景**: 适合中等波动市场，追求稳定盈利

### 2. 5m_scalping（5分钟超短线剥头皮）

**文件**: `strategies/5m_scalping.yaml`

**特点**:
- 主周期: 5分钟
- 趋势判断: 15分钟定方向
- 入场精度: 1分钟
- 仓位范围: 60%-100%
- 止损范围: 0.3%-0.6% (基于5m ATR)
- 止盈目标: 0.5%-1.2%
- 盈亏比: 最低 1:2.0
- 持仓时间: 0.5-1小时

**适用场景**: 适合高频交易，快进快出，薄利多销

### 3. 1h_swing（1小时波段交易）

**文件**: `strategies/1h_swing.yaml`

**特点**:
- 主周期: 1小时
- 趋势判断: 4小时定方向
- 入场精度: 15分钟
- 仓位范围: 40%-80%
- 止损范围: 1.0%-2.0% (基于4H ATR)
- 止盈目标: 2.5%-5.0%
- 盈亏比: 最低 1:2.0
- 持仓时间: 4-8小时

**适用场景**: 适合趋势明确的市场，追求大波段利润

## 策略配置文件结构

每个策略配置文件（YAML格式）包含以下部分：

### 1. description（策略描述）

定义 AI 交易员的角色和核心风格。

**可用变量**:
- `{symbol}`: 交易标的（从 .env 读取）
- `{initial_capital}`: 初始资金（从 .env 读取）

### 2. core_principles（核心原则）

定义策略的核心交易原则，包括趋势判断、开仓原则、仓位管理、止损止盈规则等。

**可用变量**: 所有 `parameters` 中定义的变量

### 3. strict_constraints（刚性约束）

定义必须 100% 遵守的交易规则，包括交易规则、成本约束、风控约束、持仓约束等。

**可用变量**: 所有 `parameters` 中定义的变量

### 4. output_format（输出格式）

定义 AI 输出的 JSON 格式规范。

**可用变量**: 无（固定格式）

### 5. parameters（策略参数）

定义所有可配置的策略参数。

## 创建自定义策略

### 步骤 1: 复制模板

复制现有策略文件作为模板：

```bash
cp strategies/15m_trend_following.yaml strategies/my_custom_strategy.yaml
```

### 步骤 2: 修改策略参数

编辑 `strategies/my_custom_strategy.yaml`，修改以下内容：

1. **修改策略描述**:
```yaml
description: |
  你是经验丰富的加密货币交易员,仅交易{symbol},管理{initial_capital} USDT资金,核心风格:你的策略风格描述。
```

2. **修改核心原则**:
```yaml
core_principles: |
  核心原则(你的策略周期)
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  1. 趋势判断: 你的趋势判断逻辑;
  2. 开仓原则: 你的开仓条件;
  3. 仓位原则: 你的仓位管理规则;
  ...
```

3. **修改策略参数**:
```yaml
parameters:
  # 根据你的策略调整参数值
  min_position_size_pct: 60.0
  max_position_size_pct: 100.0
  min_tp_pct: 0.8
  ...
```

### 步骤 3: 使用自定义策略

在 `.env` 文件中设置：

```bash
STRATEGY_NAME=my_custom_strategy
```

### 步骤 4: 测试策略

重启程序并观察策略执行情况：

```bash
start.bat  # Windows
# 或
./start.sh  # Linux/Mac
```

## 变量使用示例

在策略配置文件中，使用 `{变量名}` 格式引用变量：

```yaml
core_principles: |
  核心原则
  1. 仓位原则: 趋势明确时高仓位({strong_trend_position_min}%~{max_position_size_pct}%),
     弱趋势降仓位({weak_trend_position_min}%~{weak_trend_position_max}%);
  2. 止损原则: 止损间距=ATR {atr_sl_multiplier_min}~{atr_sl_multiplier_max}倍,
     单次亏损控制在{max_single_loss_pct}%以内;
  3. 止盈原则: 浮盈{min_tp_pct}%~{normal_tp_pct}%全仓平仓,
     盈亏比至少1:{min_risk_reward_ratio};
```

## 注意事项

### 1. 变量命名规则

- 所有变量必须在 `parameters` 部分定义
- 变量名使用小写字母和下划线
- 基础配置变量（symbol, initial_capital, max_daily_risk_pct）从 .env 读取，会覆盖策略文件中的值

### 2. 参数合理性

- 止盈目标应大于止损距离，确保盈亏比合理
- 仓位范围应在 0-100 之间
- 持仓时间应根据策略周期合理设置
- 成本参数应符合交易所实际费率

### 3. 策略一致性

- 确保策略描述、核心原则、刚性约束之间逻辑一致
- 参数值应与策略周期相匹配（如5分钟策略的止损应小于1小时策略）

### 4. 文件格式

- 使用 UTF-8 编码
- YAML 格式要求严格的缩进（使用空格，不要使用 Tab）
- 多行文本使用 `|` 符号

## 常见问题

### Q1: 如何查看当前使用的策略？

查看 `.env` 文件中的 `STRATEGY_NAME` 配置，或查看程序启动日志。

### Q2: 策略切换后何时生效？

需要重启程序才能生效。正在运行的持仓不受影响，新的交易决策会使用新策略。

### Q3: 可以同时运行多个策略吗？

当前版本不支持。如需同时运行多个策略，需要启动多个程序实例，每个实例使用不同的 `.env` 配置。

### Q4: 策略文件损坏怎么办？

程序启动时会验证策略文件。如果文件损坏，会显示错误信息。可以从 `strategies/` 目录中的示例文件恢复。

### Q5: 如何备份自定义策略？

将 `strategies/` 目录下的自定义 YAML 文件备份到安全位置。建议使用 Git 进行版本管理。

## 高级用法

### 动态参数计算

策略参数支持在运行时动态计算。例如，可以根据市场波动率调整止损距离：

```yaml
parameters:
  # ATR倍数会根据市场波动自动调整
  atr_sl_multiplier_min: 0.5
  atr_sl_multiplier_max: 1.0
```

### 策略组合

虽然不能同时运行多个策略，但可以创建组合策略，在不同市场条件下使用不同的参数：

```yaml
core_principles: |
  1. 趋势判断: 
     - 强趋势: 使用{strong_trend_position_min}%仓位，止盈{strong_trend_tp_pct}%
     - 弱趋势: 使用{weak_trend_position_min}%仓位，止盈{normal_tp_pct}%
```

## 策略开发建议

1. **从现有策略开始**: 复制最接近你需求的内置策略进行修改
2. **小步迭代**: 每次只修改少量参数，观察效果后再继续调整
3. **回测验证**: 使用历史数据验证策略有效性（需要自行实现回测功能）
4. **风险控制**: 始终保持严格的风控参数，避免过度激进
5. **记录调整**: 使用 Git 或文档记录每次策略调整及其效果
