# AI OKX Trader 开发实现方案

## 一、技术架构设计

### 1.1 技术栈选型

#### 后端核心
- **语言**: Python 3.11+
- **异步框架**: asyncio + aiohttp
- **定时任务**: APScheduler
- **数据库**: 
  - PostgreSQL (交易记录、历史数据持久化)
  - Redis (会话历史、实时缓存)
- **API客户端**: ccxt (统一交易所接口)
- **AI接口**: OpenAI Python SDK (GPT-4o)

#### 数据处理
- **数据分析**: pandas, numpy
- **技术指标**: ta-lib (可选，用于预计算指标)
- **数据验证**: pydantic

#### 运维监控
- **日志**: loguru
- **监控**: prometheus-client
- **告警**: 企业微信/钉钉 webhook
- **配置管理**: python-dotenv

#### 测试
- **单元测试**: pytest
- **Mock测试**: pytest-mock
- **异步测试**: pytest-asyncio

## 三、核心模块设计

### 3.1 数据模型 (data/models.py)

```python
from pydantic import BaseModel, Field
from typing import List, Optional, Literal
from datetime import datetime

class KLine(BaseModel):
    """K线数据模型"""
    timestamp: int
    open: float
    high: float
    low: float
    close: float
    volume: float

class Position(BaseModel):
    """持仓信息模型"""
    has_position: bool
    entry_price: Optional[float] = None
    size_usdt: Optional[float] = None
    current_pnl_pct: Optional[float] = None
    entry_time: Optional[datetime] = None

class KeyLevels(BaseModel):
    """关键价位模型"""
    supports: List[float] = Field(default_factory=list)
    resistances: List[float] = Field(default_factory=list)

class MarketData(BaseModel):
    """市场数据输入模型"""
    symbol: str
    current_price: float
    latest_klines: dict[str, List[float]]  # {"5m": [...], "15m": [...], "1h": [...]}
    position: Position
    key_levels: KeyLevels
    capital: float
    max_daily_risk_pct: float

class AIDecision(BaseModel):
    """AI决策输出模型"""
    d: Literal["long", "close", "wait"]  # decision
    s: Optional[int] = None              # size percentage
    e: Optional[float] = None            # entry price
    sl: Optional[float] = None           # stop loss
    tp: Optional[List[float]] = None     # take profit targets
    r: str                                # reason
```

### 3.7 主控制器 (core/controller.py)

**职责**:
- 协调所有模块
- 执行完整的交易循环
- 异常处理与恢复

**核心流程**:
```python
class MainController:
    async def run_cycle(self):
        """单次5分钟循环"""
        # 1. 获取市场数据
        market_data = await self.data_fetcher.get_incremental_data("BTC/USDT")
        
        # 2. AI决策
        decision = await self.ai_agent.make_decision(market_data)
        
        # 3. 风控验证
        if not self.risk_manager.validate_decision(decision, market_data):
            logger.warning("Decision rejected by risk manager")
            return
        
        # 4. 执行交易
        if decision.d == "long":
            await self.executor.execute_long(decision, market_data.capital)
        elif decision.d == "close":
            await self.executor.execute_close(decision.r)
        
        # 5. 保存历史
        await self.memory_manager.save_message("assistant", decision.model_dump_json())
        
        # 6. 记录日志
        await self.monitor.log_cycle(decision, market_data)
```

### 4.1 PostgreSQL 表结构

```sql
-- 交易记录表
CREATE TABLE trades (
    id SERIAL PRIMARY KEY,
    timestamp TIMESTAMP NOT NULL,
    symbol VARCHAR(20) NOT NULL,
    side VARCHAR(10) NOT NULL,  -- 'long' or 'close'
    entry_price DECIMAL(18, 8),
    exit_price DECIMAL(18, 8),
    size_usdt DECIMAL(18, 8),
    pnl_usdt DECIMAL(18, 8),
    pnl_pct DECIMAL(10, 4),
    stop_loss DECIMAL(18, 8),
    take_profit JSONB,  -- 数组存储
    reason TEXT,
    ai_decision JSONB,  -- 完整AI输出
    created_at TIMESTAMP DEFAULT NOW()
);

-- K线历史表 (可选,用于回测)
CREATE TABLE klines (
    id SERIAL PRIMARY KEY,
    symbol VARCHAR(20) NOT NULL,
    timeframe VARCHAR(10) NOT NULL,
    timestamp BIGINT NOT NULL,
    open DECIMAL(18, 8),
    high DECIMAL(18, 8),
    low DECIMAL(18, 8),
    close DECIMAL(18, 8),
    volume DECIMAL(18, 8),
    UNIQUE(symbol, timeframe, timestamp)
);

-- 风控日志表
CREATE TABLE risk_logs (
    id SERIAL PRIMARY KEY,
    timestamp TIMESTAMP NOT NULL,
    event_type VARCHAR(50),  -- 'daily_limit_hit', 'consecutive_loss', etc.
    details JSONB,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 系统状态表
CREATE TABLE system_state (
    id SERIAL PRIMARY KEY,
    key VARCHAR(100) UNIQUE NOT NULL,
    value JSONB,
    updated_at TIMESTAMP DEFAULT NOW()
);
```

## 七、关键技术难点与解决方案

### 7.1 会话历史管理
**难点**: 保持20轮对话,避免token超限
**方案**: 
- Redis存储完整历史
- 每次调用前裁剪到20轮
- 定期备份到PostgreSQL

### 7.2 止损止盈执行
**难点**: OKX现货不支持原生止损单
**方案**:
- 使用限价单模拟止损
- 实时监控价格触发
- 触发后立即市价平仓

### 7.3 关键价位计算
**难点**: 自动识别支撑/压力位
**方案**:
- 简化版: 取最近N根K线的高低点
- 进阶版: Pivot Points / Fibonacci回撤
- 由AI自主判断,不强制依赖

### 7.4 AI输出稳定性
**难点**: GPT可能输出非JSON格式
**方案**:
- 强化系统提示词 (只输出JSON)
- 使用 `response_format={"type": "json_object"}`
- 多重解析尝试 + 异常降级

### 7.5 网络异常处理
**难点**: API调用失败、超时
**方案**:
- 所有API调用加重试机制 (tenacity)
- 超时后跳过本轮,等待下一轮
- 关键错误触发告警

## 八、安全与风控

### 8.1 API密钥安全
- 使用环境变量,不提交到Git
- 服务器使用只读权限API (仅交易,不提现)
- 定期轮换密钥

### 8.2 资金安全
- 设置OKX API交易权限白名单
- 单日最大风险3%硬编码
- 异常情况自动停止交易

### 8.3 代码安全
- 所有外部输入验证 (pydantic)
- SQL注入防护 (使用ORM/参数化查询)
- 敏感日志脱敏
