# AI OKX Trader - 简化开发方案

## 一、简化原则

基于OKX官方示例代码,保持简单实用:
- ✅ 使用 `python-okx` SDK (官方推荐)
- ✅ 去掉复杂的监控组件 (prometheus等)
- ✅ 数据库和Redis设为可选
- ✅ 专注核心功能:数据获取 → AI决策 → 交易执行

## 六、数据存储方案

### 方案A: 纯内存 (最简单)
- 会话历史: Python列表 (最近20轮)
- 交易记录: 写入日志文件
- 优点: 零依赖,简单
- 缺点: 重启丢失历史

### 方案B: 本地文件 (推荐)
- 会话历史: JSON文件
- 交易记录: CSV文件
- 优点: 简单,可持久化
- 缺点: 并发性能差

### 方案C: Redis (可选)
- 会话历史: Redis List
- 交易记录: 写入日志+可选数据库
- 优点: 性能好,可扩展
- 缺点: 需要额外服务

**建议: 先用方案B,需要时升级到方案C**
