# 手动设置止损止盈指南

## 使用场景

当您遇到以下情况时，需要手动设置止损止盈：

1. **手动买入后重启程序**：程序无法自动获取止损止盈信息
2. **持仓状态文件丢失**：`logs/position_state.json` 文件不存在或损坏
3. **需要修改现有止损止盈**：调整风控参数

## 方式1：使用手动设置脚本（推荐）

### 步骤1：运行脚本

```bash
python scripts/set_stop_loss.py
```

## 方式2：手动编辑JSON文件

### 步骤1：创建文件

在项目根目录下创建 `logs/position_state.json` 文件：

```bash
mkdir -p logs
touch logs/position_state.json
```

### 步骤2：编辑文件内容

```json
{
  "symbol": "BTC-USDT",
  "entry_price": 3.4085,
  "size_btc": 0.12345678,
  "stop_loss_price": 3.3800,
  "take_profit_price": 3.4396
}
```

**字段说明**：
- `symbol`: 交易标的（必须与配置一致）
- `entry_price`: 入场价格
- `size_btc`: 持仓数量（基础货币，如BTC数量）
- `stop_loss_price`: 止损价格
- `take_profit_price`: 止盈价格（第一止盈位）

### 步骤3：重启交易程序

程序将自动从文件恢复。

## 方式3：让系统自动计算（不推荐）

如果您信任系统的自动计算，可以在 `logs/position_state.json` 中添加标志：

```json
{
  "symbol": "BTC-USDT",
  "entry_price": 3.4085,
  "size_btc": 0.12345678,
  "auto_calculate": true
}
```

系统将根据以下规则自动计算：

- **止损空间** = 0.35倍1H ATR
- **止损价** = 入场价 - 止损空间
- **止盈1** = 入场价 + 1.1倍止损空间
- **止盈2** = 入场价 + 2.0倍止损空间

**注意**：自动计算可能不符合您的实际入场逻辑，建议使用手动设置。

## 止损止盈计算建议

### 基于策略规则

根据新的5分钟超短线策略：

1. **止损空间** = 0.3-0.4倍1H ATR
2. **盈亏比** ≥ 1.1
3. **止盈设置**：
   - TP1 = 入场价 + 1.1倍止损空间（平50%）
   - TP2 = 入场价 + 2.0倍止损空间（平剩余）

### 计算示例

假设：
- 入场价：3.4085
- 1H ATR：0.0815
- 止损空间：0.35 × 0.0815 = 0.0285

计算结果：
- **止损价** = 3.4085 - 0.0285 = **3.3800**
- **止盈1** = 3.4085 + 1.1 × 0.0285 = **3.4396**
- **止盈2** = 3.4085 + 2.0 × 0.0285 = **3.4797**
- **盈亏比** = 0.0311 / 0.0285 = **1.09** ✓

## 常见问题

### Q1: 如何查看当前1H ATR？

运行以下命令查看技术指标：

```python
from src.data.okx_client import OKXClient
from src.indicators.ta_calculator import TACalculator

client = OKXClient()
calculator = TACalculator()

klines = client.get_klines("BTC-USDT", "1H", 30)
df = calculator.calculate_indicators(klines, "1H")
print(f"当前1H ATR: {df['atr'].iloc[-1]}")
```

### Q2: 止损止盈设置后可以修改吗？

可以。重新运行 `python scripts/set_stop_loss.py` 即可覆盖现有设置。

### Q3: 如果不设置止损止盈会怎样？

**非常危险！** 程序会拒绝启动价格监控，您的持仓将没有任何风控保护。

### Q4: 手动买入后忘记设置止损止盈怎么办？

程序启动时会检测到持仓并提示您设置。按照提示运行脚本即可。
