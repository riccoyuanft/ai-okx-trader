# 策略参数配置说明

## 概述

本文档说明AI交易系统中所有可配置的策略参数，帮助您根据市场情况和风险偏好灵活调整交易策略。

## 配置方式

所有策略参数都通过 `.env` 文件配置，修改后需要重启交易程序才能生效。

参考 `.env.strategy.example` 文件查看完整配置示例。

---

## 参数详解

### 1. 仓位控制参数

| 参数名 | 默认值 | 说明 | 影响 |
|--------|--------|------|------|
| `MIN_POSITION_SIZE_PCT` | 20.0 | 最小仓位比例(%) | 控制单笔交易最小资金占用 |
| `MAX_POSITION_SIZE_PCT` | 70.0 | 最大仓位比例(%) | 控制单笔交易最大资金占用 |
| `DEFAULT_POSITION_SIZE_PCT` | 40.0 | 默认仓位比例(%) | AI建议的默认开仓比例 |

**调整建议**：
- **保守策略**：15%-50%，默认30%
- **激进策略**：30%-80%，默认50%
- **当前设置**：20%-70%，默认40%（激进求稳平衡）

**影响分析**：
- 提高仓位上限 → 单笔收益增加，但风险也增加
- 降低仓位上限 → 风险降低，但收益也降低
- 调整最小仓位 → 影响AI是否愿意在低确定性机会开仓

---

### 2. 均线周期参数

| 参数名 | 默认值 | 说明 | 影响 |
|--------|--------|------|------|
| `MA_PERIOD_FAST` | 5 | 快速均线周期 | 短期趋势判断 |
| `MA_PERIOD_MID` | 10 | 中速均线周期 | 中期趋势判断 |
| `MA_PERIOD_SLOW` | 20 | 慢速均线周期 | 长期趋势判断 |

**调整建议**：
- **更灵敏趋势**：3-7-15（捕捉更多短期机会，但假信号增加）
- **更稳定趋势**：7-14-30（减少假信号，但反应较慢）
- **当前设置**：5-10-20（经典短周期均线组合）

**影响分析**：
- 缩短周期 → 入场机会增加，但假突破风险增加
- 延长周期 → 趋势更可靠，但错过短期机会
- 趋势判定条件：MA_FAST > MA_MID > MA_SLOW

---

### 3. 盈亏比与止损参数

| 参数名 | 默认值 | 说明 | 影响 |
|--------|--------|------|------|
| `MIN_RISK_REWARD_RATIO` | 1.2 | 最低盈亏比要求 | 入场质量控制 |
| `MIN_STOP_LOSS_ATR_MULTIPLIER` | 0.5 | 止损空间最小倍数(相对1H ATR) | 止损宽度控制 |

**调整建议**：
- **降低入场门槛**：盈亏比1.0，ATR倍数0.3（增加入场机会）
- **提高质量要求**：盈亏比1.5，ATR倍数0.7（减少低质量交易）
- **当前设置**：盈亏比1.2，ATR倍数0.5（平衡风险收益）

**影响分析**：
- 降低盈亏比要求 → 入场机会增加，但平均收益质量下降
- 提高盈亏比要求 → 交易质量提升，但机会减少
- 增加ATR倍数 → 止损更宽，减少被扫出，但单笔风险增加

**计算示例**：
```
假设：
- 入场价：100
- 止损价：98（风险2%）
- 盈亏比要求：1.2

则止盈价至少为：100 + 2% * 1.2 = 102.4
```

### 4. 移动止损规则参数

| 参数名 | 默认值 | 说明 | 影响 |
|--------|--------|------|------|
| `TRAILING_STOP_TRIGGER_PCT` | 0.5 | 移动止损触发门槛(价格涨幅%) | 控制调整频率 |
| `TRAILING_STOP_ATR_MULTIPLIER` | 0.5 | 移动止损安全间距(相对5m ATR) | 控制止损距离 |
| `TRAILING_STOP_MIN_DISTANCE_PCT` | 0.3 | 移动止损最小间距(%) | 最低安全距离 |

**调整建议**：
- **更宽松移动止损**（减少被扫出）：
  - 触发门槛：0.8%
  - ATR倍数：0.7
  - 最小间距：0.5%
- **更紧密移动止损**（锁定更多利润）：
  - 触发门槛：0.3%
  - ATR倍数：0.3
  - 最小间距：0.2%
- **当前设置**：平衡保护与利润锁定

**影响分析**：
- 降低触发门槛 → 止损更频繁上移，锁定利润更快，但易被正常回调扫出
- 提高ATR倍数 → 止损距离更远，不易被扫出，但回撤空间增加
- 增加最小间距 → 安全性提升，但可能错过部分利润

**工作原理**：
1. 价格上涨≥触发门槛时，才允许上移止损
2. 止损价与当前价格间距必须≥ATR倍数*5m_ATR
3. 止损价与当前价格间距必须≥最小间距百分比
4. 止损价必须低于最近支撑位
5. 只能向上调整，不能向下调整

### 5. RSI阈值参数

| 参数名 | 默认值 | 说明 | 影响 |
|--------|--------|------|------|
| `RSI_OVERBOUGHT` | 70.0 | RSI超买阈值 | 高位风险提示 |
| `RSI_OVERSOLD` | 30.0 | RSI超卖阈值 | 低位机会提示 |

**调整建议**：
- **更严格过滤**：超买65，超卖35（减少极端区域交易）
- **更宽松过滤**：超买75，超卖25（允许更多交易）
- **当前设置**：超买70，超卖30（经典RSI阈值）

**影响分析**：
- 降低超买阈值 → 更早识别高位风险，但可能错过强势行情
- 提高超卖阈值 → 更早识别低位机会，但可能在下跌途中入场

**注意**：RSI仅作为辅助参考，不作为硬性过滤条件。

---

### 6. 风险管理参数

| 参数名 | 默认值 | 说明 | 影响 |
|--------|--------|------|------|
| `MAX_CONSECUTIVE_LOSSES` | 3 | 连续亏损次数限制 | 触发保护机制的阈值 |
| `LOSS_PROTECTION_POSITION_PCT` | 30.0 | 连续亏损后的仓位限制(%) | 保护模式下的最大仓位 |

**调整建议**：
- **更严格保护**：连续亏损2次，仓位限制20%
- **更宽松限制**：连续亏损5次，仓位限制40%
- **当前设置**：连续亏损3次，仓位限制30%（适度风险保护）

**影响分析**：
- 降低连续亏损阈值 → 更早触发保护，但可能过度谨慎
- 提高仓位限制 → 恢复更快，但保护力度减弱

**工作原理**：
连续亏损达到阈值后，自动将最大仓位降低至保护值，直到盈利后重置。

---

### 7. 入场价格容差

| 参数名 | 默认值 | 说明 | 影响 |
|--------|--------|------|------|
| `ENTRY_PRICE_TOLERANCE_PCT` | 2.0 | 入场价格相对当前价格的最大偏离(%) | 限价单价格控制 |

**调整建议**：
- **更严格价格控制**：1.0%（减少滑点，但成交率降低）
- **更宽松价格容差**：3.0%（提高成交率，但滑点增加）
- **当前设置**：2.0%（适度价格容差）

**影响分析**：
- 降低容差 → 成交价格更优，但可能错过机会
- 提高容差 → 成交率提升，但可能以较差价格成交

## 参数调整场景

### 场景1：市场波动加剧，频繁被止损扫出

**问题**：价格正常波动就触发止损，导致盈利单被过早平仓。

**解决方案**：放宽移动止损间距
```env
TRAILING_STOP_ATR_MULTIPLIER=0.7
TRAILING_STOP_MIN_DISTANCE_PCT=0.5
MIN_STOP_LOSS_ATR_MULTIPLIER=0.7
```

---

### 场景2：入场机会太少，长期无交易

**问题**：AI判断入场条件过于严格，错过大量机会。

**解决方案**：降低入场门槛
```env
MIN_RISK_REWARD_RATIO=1.0
MIN_STOP_LOSS_ATR_MULTIPLIER=0.3
MA_PERIOD_FAST=3
MA_PERIOD_MID=7
```

---

### 场景3：盈利不足，需要提高盈利质量

**问题**：交易频繁但整体盈利不佳，需要提高交易质量。

**解决方案**：提高入场标准
```env
MIN_RISK_REWARD_RATIO=1.5
MIN_POSITION_SIZE_PCT=25
MAX_POSITION_SIZE_PCT=60
MIN_STOP_LOSS_ATR_MULTIPLIER=0.7
```

---

### 场景4：连续亏损，需要加强保护

**问题**：遇到不利行情，需要降低风险敞口。

**解决方案**：收紧风控参数
```env
MAX_CONSECUTIVE_LOSSES=2
LOSS_PROTECTION_POSITION_PCT=20
MAX_POSITION_SIZE_PCT=50
MIN_RISK_REWARD_RATIO=1.3
```

## 参数调整最佳实践

### 1. 调整前准备
- 记录当前参数和交易表现
- 明确调整目标（提高入场率/降低风险/提高盈利质量）
- 只调整1-2个相关参数，避免同时大幅改动

### 2. 测试流程
- 先在模拟盘测试新参数
- 观察至少20-30笔交易的表现
- 对比调整前后的关键指标：
  - 入场频率
  - 胜率
  - 盈亏比
  - 最大回撤
  - 被止损扫出的频率

### 3. 参数优化
- 根据测试结果微调参数
- 避免频繁调整，给策略足够样本量
- 记录每次调整和对应结果

### 4. 实盘应用
- 确认模拟盘效果稳定后再用于实盘
- 实盘初期降低仓位，观察实际表现
- 持续监控并根据市场变化调整

## 参数关联性

某些参数之间存在关联，调整时需要综合考虑：

### 关联组1：盈亏比与止损空间
- `MIN_RISK_REWARD_RATIO` ↑ → 需要更大的止盈空间
- `MIN_STOP_LOSS_ATR_MULTIPLIER` ↑ → 止损更宽，需要更高的盈亏比补偿

### 关联组2：仓位与风险
- `MAX_POSITION_SIZE_PCT` ↑ → 建议提高 `MIN_RISK_REWARD_RATIO`
- `MAX_CONSECUTIVE_LOSSES` ↓ → 建议降低 `LOSS_PROTECTION_POSITION_PCT`

### 关联组3：均线与移动止损
- `MA_PERIOD_*` ↓（更灵敏）→ 建议降低 `TRAILING_STOP_TRIGGER_PCT`
- `MA_PERIOD_*` ↑（更稳定）→ 建议提高 `TRAILING_STOP_ATR_MULTIPLIER`

## 常见问题

### Q1：修改参数后多久生效？
A：需要重启交易程序才能生效。已有持仓不受影响，新的交易决策会使用新参数。

### Q2：可以在持仓期间修改参数吗？
A：可以，但不建议。建议等待当前持仓平仓后再修改参数并重启。

### Q3：如何知道参数是否合适？
A：观察以下指标：
- 入场频率是否合理（不要过于频繁或过于稀少）
- 胜率是否在40%-60%之间
- 盈亏比是否达到预期
- 是否频繁被止损扫出

### Q4：参数调整后效果不佳怎么办？
A：恢复默认参数，重新分析问题根源，小幅度调整单个参数进行测试。
