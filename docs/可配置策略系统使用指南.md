# 可配置策略系统使用指南

## 快速开始

### 1. 复制配置模板

```bash
cp .env.strategy.example .env
```

### 2. 编辑配置文件

根据您的需求修改 `.env` 文件中的策略参数。

### 3. 重启交易程序

```bash
python src/main.py
```

---

## 核心优势

### ✅ 完全可配置
所有策略参数都可以通过 `.env` 文件动态调整，无需修改代码。

### ✅ 双层同步
- **Prompt层**：AI决策逻辑自动使用最新参数
- **代码层**：风控校验和执行逻辑同步更新

### ✅ 灵活调整
根据市场情况和交易表现，随时调整策略参数。

## 典型应用场景

### 场景1：提高入场频率

**现象**：系统长时间不开仓，错过很多机会。

**调整方案**：
```env
# 降低盈亏比要求
MIN_RISK_REWARD_RATIO=1.0

# 缩短止损空间要求
MIN_STOP_LOSS_ATR_MULTIPLIER=0.3

# 使用更灵敏的均线
MA_PERIOD_FAST=3
MA_PERIOD_MID=7
MA_PERIOD_SLOW=15
```

**预期效果**：
- 入场机会增加50%-100%
- 胜率可能略有下降
- 需要更严格的止损执行

---

### 场景2：减少被止损扫出

**现象**：盈利单经常被正常回调触发止损。

**调整方案**：
```env
# 放宽移动止损触发门槛
TRAILING_STOP_TRIGGER_PCT=0.8

# 增加止损安全间距
TRAILING_STOP_ATR_MULTIPLIER=0.7
TRAILING_STOP_MIN_DISTANCE_PCT=0.5

# 放宽初始止损空间
MIN_STOP_LOSS_ATR_MULTIPLIER=0.7
```

**预期效果**：
- 被扫出频率降低60%-80%
- 单笔最大回撤增加
- 持仓时间可能延长

---

### 场景3：提高交易质量

**现象**：交易频繁但整体盈利不佳。

**调整方案**：
```env
# 提高盈亏比要求
MIN_RISK_REWARD_RATIO=1.5

# 增加止损空间要求
MIN_STOP_LOSS_ATR_MULTIPLIER=0.7

# 降低最大仓位
MAX_POSITION_SIZE_PCT=60

# 使用更稳定的均线
MA_PERIOD_FAST=7
MA_PERIOD_MID=14
MA_PERIOD_SLOW=30
```

**预期效果**：
- 交易频率降低30%-50%
- 单笔盈利质量提升
- 胜率可能提高

---

### 场景4：加强风险控制

**现象**：遇到不利行情，需要降低风险。

**调整方案**：
```env
# 降低最大仓位
MAX_POSITION_SIZE_PCT=50

# 提高盈亏比要求
MIN_RISK_REWARD_RATIO=1.3

# 更严格的连续亏损保护
MAX_CONSECUTIVE_LOSSES=2
LOSS_PROTECTION_POSITION_PCT=20
```

**预期效果**：
- 单笔风险降低
- 连续亏损后快速降低仓位
- 整体回撤控制更好

## 参数调整工作流

```
1. 识别问题
   ↓
2. 分析原因
   ↓
3. 确定调整目标
   ↓
4. 选择相关参数
   ↓
5. 模拟盘测试（20-30笔交易）
   ↓
6. 评估效果
   ↓
7. 实盘应用（降低仓位）
   ↓
8. 持续监控
```

---

## 参数调整原则

### ✅ DO（推荐做法）

1. **小步调整**：每次只调整1-2个相关参数
2. **充分测试**：模拟盘至少观察20-30笔交易
3. **记录对比**：记录调整前后的关键指标
4. **逐步应用**：实盘初期降低仓位观察
5. **持续优化**：根据市场变化动态调整

### ❌ DON'T（避免做法）

1. **频繁调整**：不要每天都改参数
2. **大幅改动**：不要同时调整5个以上参数
3. **盲目跟随**：不要照搬别人的参数
4. **忽略测试**：不要跳过模拟盘直接实盘
5. **过度优化**：不要追求完美参数

## 监控指标

### 关键指标

| 指标 | 理想范围 | 说明 |
|------|----------|------|
| 入场频率 | 每周2-5次 | 太少错过机会，太多可能过度交易 |
| 胜率 | 40%-60% | 过高可能盈亏比不足，过低需要检查策略 |
| 平均盈亏比 | ≥1.2 | 确保长期盈利的基础 |
| 最大回撤 | ≤10% | 单笔或连续亏损的最大损失 |
| 被止损扫出率 | ≤20% | 盈利单被正常波动扫出的比例 |

### 监控方法

1. **每日检查**：查看交易日志和持仓状态
2. **每周总结**：统计关键指标，对比目标
3. **每月复盘**：分析所有交易，识别改进点

## 常见问题

### Q1：参数太多，不知道从哪里开始？

**A**：按优先级调整：
1. 如果入场太少 → 调整盈亏比和均线周期
2. 如果频繁被扫出 → 调整移动止损参数
3. 如果盈利不佳 → 调整仓位和盈亏比要求

### Q2：调整参数后效果不明显？

**A**：可能原因：
- 样本量不足（至少观察20笔交易）
- 调整幅度太小（建议调整20%-30%）
- 市场环境变化（需要结合市场情况分析）

### Q3：如何判断参数是否合适？

**A**：综合评估：
- 入场频率适中（不要过于频繁或稀少）
- 胜率和盈亏比平衡（胜率40%+盈亏比1.5 或 胜率60%+盈亏比1.0）
- 回撤可控（最大回撤≤10%）
- 心理舒适（不会因为频繁被扫出而焦虑）

### Q4：可以针对不同标的使用不同参数吗？

**A**：可以。建议：
- 为不同标的创建不同的 `.env` 文件
- 使用不同的配置文件启动多个实例
- 根据标的特性（波动率、流动性）调整参数

## 进阶技巧

### 1. 参数组合策略

根据市场阶段使用不同参数组合：

**趋势行情**：
```env
MA_PERIOD_FAST=5
MIN_RISK_REWARD_RATIO=1.5
TRAILING_STOP_TRIGGER_PCT=0.3
```

**震荡行情**：
```env
MA_PERIOD_FAST=7
MIN_RISK_REWARD_RATIO=1.0
TRAILING_STOP_ATR_MULTIPLIER=0.7
```

### 2. 动态调整策略

根据交易表现动态调整：

**连续盈利后**：
- 可适当提高仓位上限
- 放宽入场条件

**连续亏损后**：
- 自动触发保护机制（已内置）
- 可手动进一步收紧参数

### 3. A/B测试

同时运行两组参数对比效果：
- 实例A：保守参数（模拟盘）
- 实例B：激进参数（模拟盘）
- 选择表现更好的应用于实盘

## 技术实现

### 配置加载流程

```
.env文件
  ↓
Settings类（pydantic）
  ↓
Prompt动态生成（AI决策层）
  ↓
代码校验逻辑（执行层）
```

### 参数生效时机

- **Prompt参数**：每次AI决策时实时读取
- **代码参数**：程序启动时加载，重启后生效

### 参数验证

所有参数都经过类型检查和范围验证，确保配置安全。
